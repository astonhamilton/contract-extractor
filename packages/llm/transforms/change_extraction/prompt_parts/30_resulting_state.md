Resulting State Domain:
- Question answered: what post-change state is explicitly established?
- resulting_state.answer should describe only the resulting state that the change artifact clearly states.
- resulting_state.citations should cite the clause that states the post-change effect.

Decision steps:
1. Look for language that states the resulting term, pricing, scope, control/process, or other operative state after the change.
2. Capture only what is explicit.
3. Keep the note narrow when the new state is only partially stated.
4. Prefer the specific substantive resulting state over boilerplate "all other terms remain in effect" language when both are present.

Guardrails:
- Do not infer a full resulting state from background context or prior documents.
- Do not invent old/new comparisons that the document does not make explicit.
- If the document only states a mechanism for future change, keep this note sparse.
- Do not use generic remaining-provisions boilerplate as the whole resulting-state answer when the amendment also explicitly states concrete new rates, dates, scope, or obligations.
